# 数据类型与检测

# 知识图谱

本文涉及到的知识点：

# JavaScript 中的数据类型

数据类型分为：基本数据类型、引用数据类型

  • 基本类型：String、Number、Boolean、Null、Undefined、Symbol

  • 引用类型：对象类型也称引用类型，它包含很多子类，如：Array、Function、Object、Date、RegExp

# 数据类型与堆栈

堆和栈都是运行时内存中分配的一个数据区，因此也被称为堆区和栈区；二者存储的数据类型和处理速度不同，如下：

  • 堆（heap）用于复杂数据类型（引用类型）分配空间，例如数组对象、object对象；它是运行时动态分配内存的，因此存取速度较慢。

  • 栈（stack）中主要存放一些基本类型的变量和对象的引用，其优势是存取速度比堆要快，并且栈内的数据可以共享，但缺点是存在栈中的数据大小与生存期必须是确定的，缺乏灵活性，

基本类型存储在栈内存中，一个变量复制另一个变量时，会在栈中创建一个新值，然后把值复制给新变量，新变量的值发生改变时不会影响到被复制的变量的值。

引用类型将数据存在堆内存中，而变量作为指针存放在栈内存中，当我们访问引用数据时，先从栈内存中获取指针，通过指针在堆内存中找到对应的数据

# 数据类型的检测方式

数据类型常见的检测方式如下：

  • typeof
  • instanceof
  • constructor
  • Object.prototype.toString.call()

# typeof

typeof 123            // "number"
typeof '123'          // "string"
typeof true           // "boolean"
typeof undefined      // "undefined"
typeof null           // "object"
typeof NaN            // "number"
typeof []             // "object"
typeof {}             // "object"
typeof function(){}   // "function"
typeof new Date       // "object"

注意：

typeof操作符只可以对除null以外的基本类型做出正确的判断，对于引用数据类型，检测出来都是object，无法判断具体属于哪种对象

# instanceof

instanceof 是用来判断 A 是否为 B 的实例，表达式为：A instanceof B ，如果 A 属于 B 的实例，则返回 true ，否则返回 false

console.log("1" instanceof String)             // false
console.log(1 instanceof Number)               // false
console.log(true instanceof Boolean)           // false
console.log([] instanceof Array)               // true
console.log([] instanceof Object)              // true
console.log(function(){} instanceof Function)  // true
console.log({} instanceof Object)              // true

注意：

  • instanceof 只能用来判断两个对象是否属于实例关系， 而不能判断对象实例具体属于哪种类型

  • instanceof 无法对原始类型进行判断，所以一般都是在 typeof 判断为 object 时才使用 instanceof

# constructor

constructor可以打印出实例所属的类，用法：(实例).constructor

('abc').constructor === String           // true
(1).constructor === Number               // true
(true).constructor === Boolean           // true
([]).constructor === Array               // true
([]).constructor === Object              // false
({}).constructor === Object              // true
(function(){}).constructor === Function  // true
(function(){}).constructor === Object    // false

注意：

  • null、undefined因为没有constructor存在，因此这这两个不能通过这种方式来判断

  • 如果重写对象的prototype，原有的constructor会丢失，判断结果会发生改变

# Object.prototype.toString.call()

每个Object上都有一个toString()原型方法，调用该方法会默认返回当前对象的[[Class]]属性的值，[[Class]]是一个内部属性，所有对象（原生对象和宿主对象）都拥有该属性，其格式为[object Xxx] ，如数组：[object Array]

Object直接调用toString()就能返回[object Object]，但是其他对象需要通过call/aplly才能返回正确的类型信息

Object.prototype.toString.call('abc')          // "[object String]"
Object.prototype.toString.call(123)            // "[object Number]"
Object.prototype.toString.call(true)           // "[object Boolean]"
Object.prototype.toString.call(null)           // "[object Null]"
Object.prototype.toString.call(undefined)      // "[object Undefined]"
Object.prototype.toString.call([])             // "[object Array]"
Object.prototype.toString.call({})             // "[object Object]"
Object.prototype.toString.call(function(){})   // "[object Function]"

我们会发现，Object.prototype.toString.call()是可以用来正确检测js所有的数据类型的
