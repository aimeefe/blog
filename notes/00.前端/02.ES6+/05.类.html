# Class

# Class 声明

类声明以class关键字开始，其后是类的名称；剩余部分的语法看起来就像对象字面量中的方法简写，方法之间不需要逗号

举例：

// 声明一个类
class Person {
  // 等价于 ES5 中的构造器，类声明允许在其中使用`constructor`方法名称直接定义一个构造器，而无需先定义一个函数再把它当做构造器使用
  constructor(name) {
    // 一些自有属性...，该属性会出现在实例上而不是原型上，只能在类的构造器或方法内部进行创建
    this.name = name
  }

  // 等价于 Person.prototype.sayName
  sayName() {
    console.log(this.name);
  }
}

// 使用
let person = new Person('lily')
person.sayName()  // lily
console.log(person instanceof Person)  // true

注意：

  1. 类的自有属性会出现在实例上，这些自有属性只能在类的constructor中进行创建
  2. 类的方法都会出现在原型prototype上，可以通过Object.assign()一次性的添加多个方法到原型上，如：
class Person {
  constructor(name) {
    this.name = name
  }

  sayName() {
    console.log('name')
  }
}

Object.assign(Person.prototype, {
  sayHi() {
    console.log('hi')
  },
  sayAge() {
    console.log('age')
  }
})

# Class 表达式

与函数一样，类也可以使用表达式的形式定义，如：

let PersonClass = class Person {
  constructor(name) {
    this.name = name
  }
  sayName() {
    console.log(this.name)
  }
}

// ---------- 等价于 ----------
let PersonClass = (function() {
  'use strict'
  const Person = function(name) {
    // ...
  }
  return Person
}())

// 使用
let person = new Person('lily')  //抛出错误：Person is not defined，这是因为外部不存在 Person 绑定
let person2 = new PersonClass('lily')
console.log(person2)  // { name: 'lily' }

上例使用具名表达式定义了一个类，这个类的名称是PersonClass而不是Person，Person只能在PersonClass内部使用来指代当前类，如果Person在内部没有用到的话，可以省略，采用这面这种普通表达式定义类，如：

let Person = class {
  constructor(name) {
    this.name = name
  }
  sayName() {
    console.log(this.name)
  }
}

// 使用
let person = new Person('lily')
console.log(person)  // {name: 'lily'}

# Class 表达式的应用

# 立即执行的类

使用Class表达式可以写出一个立即执行的类来，如：

let person = new class {
  constructor(name) {
    this.name = name
  }
  sayName() {
    console.log(this.name)
  }
}('lily')

person.sayName()   // lily

如上，person是一个立即执行的Class的实例

# 作为参数传入函数

function createPerson(classDef) {
  return new classDef('lily')
}

let person = createPerson(
  class {
    constructor(name) {
      this.name = name
    }
    sayName() {
      console.log(this.name)
    }
  }
)

console.log(person)  // {name: lily}
person.sayName()  // lily

# Class 特性

  1. 类声明不会被提升，与let相似，因此在程序的执行到达声明处之前，类会存在暂时性死区
let person = new Person('lily')

class Person {
  constructor(name) {
    this.name = name
  }
  sayName() {
    console.log(this.name)
  }
}

console.log(person)  // 抛出错误：Cannot access 'Person' before initialization

  1. 类声明中所有代码自动运行在严格模式下，无法退出严格模式

  2. 类的所有方法都是不可枚举的，而自定义类型必须通过Object.defineProperty()才能将方法改变为不可枚举

//...
let person = new Person('lily') 
console.log(person)  // {name: 'lily'}

for (const item in person) {
  console.log(item)
  // ES6 类输出：name
  // 自定义类型输出：name, sayName
}

  1. 类的所有方法内部都没有[[Construct]]，因此使用new来调用他们会抛出错误

  2. 调用类构造器时不使用new会抛出错误

//...
ler person = Person()  // 抛出错误：Class constructor Person cannot be invoked without 'new'
  1. 试图在类的方法内部重写类名会抛出错误
class Person {
  // ...
}

// -------- 等价于 -------- 
let Person = (function(){
  'use strict'
  const Person = function(name) {
    // ...
  }
}())

// `ES6`中，类名是通过`const`定义的，因此重写是会抛出错误的，但是只是在内部不可重写，在类的外部是可以的，如：
Person = '123'
console.log(Person)  // 123
  1. this的指向

类的方法内部如果含有this，它默认会指向类的实例

class Person {
  constructor(name) {
    this.name = name
  }

  sayName() {
    // this 指向类的实例
    console.log(this.name)  
  }
}

let person = new Person('lily')   // name 赋值操作，set 被调用，val 为 lily
person.sayName()   // lily

注意：如果单独使用该方法时，会抛出错误，如：

class Person {
  constructor(name) {
    this.name = name
  }

  sayName() {
    // 该方法单独使用时，由于 Class 自动运行在严格模式下，会抛出错误：Cannot read property 'name' of undefined
    console.log(this.name)
  }
}

const person = new Person('lily')
const { sayName } = person
sayName()  // 抛出错误

解决的办法：在构造方法constructor中绑定this

class Person {
  constructor(name) {
    this.name = name
  + this.sayName = this.sayName.bind(this)
  }

  sayName() {
    console.log(this.name)
  }
}

const person = new Person('lily')
console.log(person)  // Person { name: 'lily', sayName: [Function: bound sayName] }

const { sayName } = person
sayName()  // lily

# constructor

constructor是类的默认方法，通过new命令生成对象实例时，会自动调用它

一个类必须要有constructor方法，如果没有显示定义，那么一个空的constructor方法会被默认添加：constructor(){}，constructor()方法默认返回实例对象（即 this），也可以指定另外一个对象，如：

class Person {
  constructor(name) {
    this.name = name
    return Object.create(null)
  }
}

// 使用
let person = new Person('lily')
console.log(person, person.name)  // {}, undefined
console.log(person instanceof Person)  // false

上述代码中，constructor返回了一个新的对象，结果导致了实例对象不是Person类的实例了

# 实例属性的新写法

实例属性除了定义在constructor的this上，也可以定义在类的最顶层，如：

class Foo {
  bar = 'hello';
  baz = 'world';

  constructor() {
    // 需要接受参数的 可以放这里...
  }
}

把所有实例对象自身的属性都定义在类的头部，这样看上去比较整齐，一眼就能看出这个类有哪些实例属性

# getter 和 setter

类的内部可以使用get和set关键字来对某个属性设置存值函数和取值函数，拦截该属性的存取行为

class Person {
  constructor(name) {
    this.name = name
  }

  get name() {
    console.log(`name 取值操作，get 被调用`)
    return 'sasa'
  }

  set name(val) {
    console.log(`name 赋值操作，set 被调用，val 为 ${val}`);
  }
}


let person = new Person('lily')   // name 赋值操作，set 被调用，val 为 lily
console.log(person.name) // name 取值操作，get 被调用， 取到的值是 sasa

# static 定义静态属性和方法

# 静态属性

静态属性指的是Class本身的属性，即Class.propName而不是定义在实例对象this上的属性，因此它只能供Class本身使用，实例是访问不到它的

# 旧写法

class Person {
  sayName() {
    console.log(Person.staticName)   // lily
  }
}

// 定义一个静态属性`  
Person.staticName = 'lily'

let person = new Person()
person.sayName()

# 新写法

通过static关键字定义

class Person {
  // 定义一个静态属性
  static staticName = 'lily'
  sayName() {
    console.log(Person.staticName)   // lily
  }
}

let person = new Person()
person.sayName()

# 静态方法

静态方法通过在方法前加static关键字来实现，该方法不会被实例继承，而是只供类来调用。静态方法常用于为一个应用程序创建工具函数

class Person {
  // 定义一个静态方法
  static sayName() {
    console.log('lily')
  }
}

Person.sayName()   // lily
// let person = new Person()
// person.sayName()  // 抛出错误：person.sayName is not a function

注意：如果静态方法包含this，那么这个this的指向是类而不是实例，如：


class Person {
  static name = 'lily'
  constructor() {
    this.name = 'sasa'
  }
  // 定义一个静态方法
  static sayName() {
    // 静态方法的 this 指向类而非实例，相当于 Person.name
    console.log(this.name)  // lily
  }
}

Person.sayName()

如上，静态方法中的this指向的是Person类，因此打印出的name值是lily

# new.target

new.target一般应用在构造函数中，用来判断构造函数被调用的方式（是否通过new调用等）

function Person() {
  if (new.target !== undefined) {
    this.name = 'lily'
  } else {
    throw new Error('必须使用 new 命令生成实例')
  }
}

new.target会返回当前的Class，如果子类继承了父类，那么new.target会返回子类，因此我们可以用来控制某些构造函数是否可以被实例化，如：

class Person {
  constructor() {
    // console.log(new.target === Student) // true
    if (new.target !== Person) {
      throw new Error('我只能被用来继承，不能被用来实例化')
    }
  }
}

class Student extends Person { }

let Student = new Student()

# Class 的继承

# extends

Class之间可以通过extends关键字实现继承：

class Student extends Person {}

这段代码中Student通过extends继承了Person类所有的属性和方法，这比ES5通过修改原型链实现继承要清晰和方便的多，接下来我们就可以在Person内部去定制自己的类了

# super

super有两种用法，一种是作为函数使用，另一种是作为对象使用

# super 作为函数使用

super作为函数调用时，指向父类的构造函数constructor

class Person {
  constructor(name) {
    this.name = name
    this.age = 18
  }
}

class Student extends Person {
  constructor(name) {
    super(name)
  }
}

// 使用
let stu = new Student('lily', '四年级')
console.log(stu)  // Student {name: "lily", age: 18}

注意：

  1. 子类要继承父类的this对象，子类本身是没有的。因此子类必须在constructor方法中调用一次super方法，否则创建实例的时候会抛出错误；如果子类没有显式的定义constructor，super方法会被默认添加以确保每个子类都会有一个constructor方法，如下：
constructor(args){
  super(...args)
}

ES5 继承中的 this 和 ES6 继承中的 this：

  • ES5 的继承：先创造子类的实例对象this，然后将父类的方法添加到this上（Parent.apply(this)）

  • ES6 的继承：先创造父类的实例对象this（所以必须先调用 super 方法），然后再用子类的构造函数去修改this

  1. 子类的constructor中，只有调用super之后才可以使用this关键字，否则会抛出错误。这是因为子类实例的构建是基于对父类实例的加工，只有super方法才能返回父类的实例
class Person {//...}

class Student extends Person {
  constructor(name, grade) {
    this.grade = grade  // 这里的 this 先于 super 使用了，因此会抛出错误
    super(name)
  }
}

let stu = new Student('lily', '四年级')
  1. super只能用在子类的构造函数中，用在其他地方会报错

# super 作为对象使用

super作为对象时：

  • 普通方法中：指向父类的原型对象
  • 静态方法中：指向父类

🍃普通方法举例：

class Person {
  constructor() {
    this.name = 'lily'
  }
  getName() {
    return this.name
  }
}

class Student extends Person {
  constructor() {
    super()
    this.name = 'sasa'
  }
  sayHi() {
    // super 作为对象使用时，指向父类的原型
    // super 调用父类方法时，方法内部的 this 指向当前的子类实例
    const name = super.getName()
    console.log(`Hi, My name is ${name}`);
    // super 作为对象使用时是无法获取父类实例的
    console.log(super.name)  // undefined
  }
}

let stu = new Student()
stu.sayHi() // Hi, My name is sasa

注意：

  1. super作为对象使用时，普通方法中指向父类原型，因此父类实例（constructor）上的属性是拿不到的

  2. 在子类普通方法中通过super调用父类的方法时，方法内部的this指向当前的子类实例

🍃静态方法举例：

class Person {
  static name = 'lily'
}

class Student extends Person {
  static sayName() {
    // static 中使用 super，指向父类
    console.log(super.name) // lily
  }
}

// 使用
Student.sayName() // lily

注意：

在子类的静态方法中通过super调用父类的方法时，方法内部的this指向当前的子类，而不是子类的实例

# 其他

由于对象总是继承其他对象的，所以可以在任意一个对象中，使用super关键字

let obj = {
  isObj(o) {
    return super.toString.call(o) === '[object Object]'
  }
}

obj.isObj({ name: 'lily' }) // true
obj.isObj(1111)  // false

# Class 的原型

ES5中，每个对象都由一个__proto__属性，指向对应构造函数的prototype属性。

而Class作为构造函数的语法糖，同时具有__proto__和Prototype属性，因此同时存在两条继承链：

  1. 子类的__proto__属性表示构造函数的继承，总是指向父类
  2. 子类prototype属性的__proto__，表示方法的继承，总是指向父类的prototype属性
class Person {
  constructor(name) {
    this.name = 111
  }
  sayName() {
    console.log(111)
  }
}

class Student extends Person {}

// 子类的 __proto__ 属性指向父类
console.log(Student.__proto__ === Person)  // true
// 子类的 prototype 的 __proto__ 属性指向父类的 prototype
console.log(Student.prototype.__proto__ === Person.prototype) // true
